Lists organize content in bullet points (<ul>) or numbered sequences (<ol>).

<ul>
    <li>Item 1</li> 
    <li>Item 2</li>
</ul>
          
<ol>
     <li>Step 1</li>

</ol>

  • Item 2

Tables

<table> organizes data into rows (<tr>) and columns (<td>).

     <tr>
        <td>Row 1, Cell 1</td>
        <td>Row 1, Cell 2</td>
     </tr>
 </table>

Transitions

Transitions enable smooth changes between property values.

.button {
     background-color: #8800a0;
     padding: 10px 20px;
     border: none;
     cursor: pointer;
     transition: background-color 0.3s;
     }
      
.button:hover {
     background-color: #c904ec;
}

Protocols

HTTP (HyperText Transfer Protocol)

HTTP is a protocol used for transferring hypertext (web pages) over the web. It is the foundation of data communication on the internet.


  • HTTP Methods:
    • GET: Retrieves data from the server.
    • POST: Sends data to the server (used for submitting forms).
    • PUT: Updates existing resources on the server.
    • DELETE: Deletes a specified resource from the server.
    • PATCH: Partially updates a resource.
    • HEAD: Similar to GET, but retrieves headers only.
    • OPTIONS: Describes the communication options for the target resource.

HTTPS (HyperText Transfer Protocol Secure)

HTTPS is the secure version of HTTP. It adds a layer of security using SSL/TLS encryption, which protects the data being transferred between the browser and the server.


  • HTTP Response Status Codes:
    • 1xx (Informational): e.g., 100 Continue (The client should proceed).
    • 2xx (Successful): e.g., 200 OK (Request was successful).
    • 3xx (Redirection): e.g., 301 Moved Permanently (Resource moved permanently).
    • 4xx (Client Error): e.g., 404 Not Found (The resource cannot be found).
    • 5xx (Server Error): e.g., 500 Internal Server Error (Unexpected condition occurred on the server).

DNS (Domain Name System)

DNS is a system that translates human-readable domain names (e.g., google.com) into IP addresses, which computers use to communicate with each other. DNS records store various types of information about domains.

DNS Record Types:


  • DNS Record Types:
    • A: Maps a domain to an IPv4 address.
    • AAAA: Maps a domain to an IPv6 address.
    • CNAME: Alias for a domain, points to another domain.
    • MX: Specifies mail servers for email reception.
    • TXT: Holds text data, often used for security (SPF, DKIM).
    • NS: Defines the authoritative name servers for a domain.
    • PTR: Maps an IP address to a domain name (reverse lookup).
    • SOA: Specifies domain zone details, like primary name server.
    • SRV: Defines locations for services like SIP or LDAP.
    • CAA: Specifies which Certificate Authorities can issue certificates for a domain.

IP (Internet Protocol)


IP is responsible for addressing and routing data packets across the internet.

  1. Determining an IP Address:

    To find the IP address of a domain (e.g., google.com or google.com.ua), you can perform a DNS lookup. Use command-line tools like ping or nslookup, or visit online services.


  2. Types of IP Addresses:
    • IPv4: e.g., 192.168.0.1 (4 octets, separated by dots).
    • IPv6: e.g., 2001:0db8:85a3:0000:0000:8a2e:0370:7334 (8 groups of 4 hexadecimal digits, separated by colons).

  3. How IP Relates to Domains:

    When you visit google.com, it resolves to an IP address like 142.250.190.46. Similarly, google.com.ua may resolve to another address like 216.58.213.174 or an IPv6 address.
